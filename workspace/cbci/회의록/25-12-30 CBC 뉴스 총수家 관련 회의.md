# 12-30 CBC 뉴스 총수家 관련 회의

<br>

## 1) 콘텐츠 필터·리스크 관리

- **긍/부정 필터링**
    - 긍정/부정 판별 결과를 기사/리스트에 **표시**(배지 등) 
    - “긍정 기사만 보기”등의 필터 제공
    - 뉴스 감성분석 모델 사용해볼것(KoELECTRA 등)
        
- **정치家 개발 시 여론조사 배제**
    - 여론조사 관련 “위험 키워드”는 **컬러 마킹**으로 경고 처리(노출은 하되 주의 표시)
- **총수家 오너리스크 범위 제한**
    - 총수家의 기사 수집 범위는 **부모·자식 중심**(오너리스크가 직접 연결될 범위까지만)
        
<br>


## 2) 데이터 품질·중복 방지

- **기사 중복 체크 강화**
    - 1차: **CID 기준 중복 제거**
    - 2차: **제목 유사/중복 필터링**(제목 기반 중복 방지 로직 추가)
        
- **캡션 표준화**
    - 캡션은 항상 `사진=연합뉴스`로 고정 처리

<br>

## 3) 관리자·운영 기능(관리자 페이지 요구사항)
- **기업 요청 페이지**를 관리자 페이지에서 생성/관리 가능하도록 구축
    
- **타임라인 누적 상태 표시**
    - 타임라인 이벤트가 **10개 이상 누적된 기업**을 관리자 리스트에서 쉽게 식별(배지/아이콘/필터)
    - 타임라인 개수를 각 기업 별로 파악할수 있게 표시할것
        

<br>


## 4) 카테고리 확장·편성 논의
- **뉴스家 진행 우선순위**
    - **총수家 → 기업家 → 정치家 → 금융家 → 스타家** 순으로 개발/운영 진행
    - 개발 후 트래픽을 파악하고, 추후 개발 순서 조정 필요
        
- **기업家 확장**
    - 총수家에서 쓰지 않는 기업(**코스닥 상위 10개** 등)은 **기업家로 편입**해 데이터 확장
        
- **정치家 구성**
    - “스타 정치인”을 **여야 균형**에 맞춰 배치(내년 총선 대비)
        
- **엔터(콘텐츠 편성)**
    - 가볼만한곳 / 드라마 편성(1회~마지막회) / 시청률 등 **엔터형 타임라인 콘텐츠** 별도 운영
        
- **경제성 타임라인(지표 누적 송고)**
    - **날씨·환율·주식**을 타임라인 형태로 축적해 송고(지표형 이벤트 누적)
        

<br>


## 5) 제목 정책(에디토리얼 룰)

- **제목 스타일 가이드 정의**
    - **의문형 / 표준형** 등 유형을 정하고, 카테고리별 기본값/금지 패턴까지 룰화(추후 CBC에서 정의)
